TOPIC 1: Daily Workflow Mapping
1

🗓️ Daily Workflow Mapping

O daily workflow mappingdefines which prompt to use at each phase of your workday. Instead of choosing prompts at random, you follow a predictable flow that maximizes productivity and quality.

🎯Ideal Daily Workflow

08:00 - Start of the Day
Open Claude Code →95% Confidence→ Study what you are going to work on today
08:30 - Planning
Plan Mode Blueprint→ Generate the blueprint of the main task of the day
09:00-12:00 - Construction
Feature Blueprint + Build With Tests→ Implement step by step
12:00 - Middle of the Day
/compact→ Compact the context if the session is long
14:00-17:00 - Continuation + Debug
Structured Bug Reportwhen you find bugs → Continue building
17:00 - End of the Day
Brutal Audit→ Review everything before final commit

💡Practical Tip

Adapt the schedule to your rhythm. The important thing is not the time, but the sequence: Study → Plan → Build → Debug → Review. This progression ensures that you always start informed and end with quality.

TOPIC 2: 95% Confidence for Complex
2

🎯 95% Confidence for Complex Tasks

O 95% ConfidenceIt is mandatory for complex tasks: changes to multiple files, changes in critical areas (auth, payments, data), or entry into unknown projects. Skipping this step on complex tasks is the #1 cause of rework.

🎯When to Use (and When Not)

USE 95% Confidence

  • ✓Task touches 3+ files
  • ✓Critical area (auth, payment, data)
  • ✓New or unknown project
  • ✓Architectural change
  • ✓Complex/intermittent bug

You DO NOT need 95%

  • ✗Rename variable
  • ✗Adjust text/CSS
  • ✗Add log/comment
  • ✗Obvious bug in 1 file
  • ✗Known repetitive task

💡Practical Tip

When in doubt, use it. The cost of 5 minutes of study is insignificant compared to the cost of 2 hours of rework for starting without understanding the context.

TOPIC 3: Plan Mode for Building
3

📋 Plan Mode for Building

O Plan ModeIt's your best friend during the construction phase. It ensures you have a map before you start navigating. Without a plan, you build in the dark and discover problems too late.

🎯Plan Mode Strategy

FLUXO: Plan Mode → Approve → Implement Step 1 → Review → Implement Step 2 → ...

Comandos chave durante building:
1. "Use ultrathink para planejar isso" → Ativa planejamento profundo
2. "Implemente apenas o passo 1" → Execucao incremental
3. "PARE. Me mostre o que fez antes de continuar" → Gate de controle
4. "Isso mudou o comportamento? Nao deveria." → Guardrail
5. "Rode os testes antes de prosseguir" → Verificacao continua

Nunca diga: "Faz tudo de uma vez"
Sempre diga: "Faz o proximo passo e para"

💡Practical Tip

If the plan has more than 5 steps, decompose it into sub-plans. A 15-step plan is just as useful as no plan at all. Keep each plan to 3-5 actionable steps.

TOPIC 4: Brutal Audit for Closing
4

🔍 Brutal Audit for Closing

O Brutal Auditis the last prompt before ending a session or making an important commit. It is your final quality gate - nothing goes into production without passing through it.

🎯Closing Ritual

Antes de encerrar esta sessao, faca:

1. Brutal Audit de tudo que foi construido/modificado hoje
2. Liste todos os arquivos modificados (git diff --name-only)
3. Para cada arquivo: ha algo que me preocupa?
4. Testes: todos passam? Ha testes novos necessarios?
5. Documentacao: CLAUDE.md precisa ser atualizado?
6. TODO list: o que ficou pendente para a proxima sessao?

Classifique o status geral:
- PRONTO PARA COMMIT: Tudo limpo, pode ir
- PRECISA AJUSTE: Pequenas correcoes antes de commitar
- NAO COMMITAVEL: Problemas serios, nao commite assim

💡Practical Tip

The item "TODO list for next session" is crucial. When you start the next session, you know exactly where you left off and what you need to do. Save this TODO in CLAUDE.md or in a session file.

TOPIC 5: Teach Me for Learning
5

📚 Teach Me for Continuous Learning

O Teach MeIt's not just for moments of doubt - it's a continuous learning strategy. Use it systematically to deepen your knowledge as you work, turning each session into an opportunity for growth.

🎯Ideal Times to Teach Me

  • • Claude used a pattern that you don't know- "Teach me the pattern [X] you just applied"
  • • An architectural decision is not obvious- "Why is this approach better than [alternative]?"
  • • A bug revealed a concept you didn't master- "Teach me [concept] so I can avoid this type of bug"
  • • End of the session with free time- "Teach me something interesting about the code we work on today"

💡Practical Tip

Keep a "learning diary" on the project. Each time Teach Me reveals something valuable, briefly note it in CLAUDE.md or a separate file. In 3 months, you will have a wealth of contextualized knowledge.

TOPIC 6: Context Clearing Strategy
6

🧹 Context Cleaning Strategy

A context cleanupIt's just as important as the prompts themselves. A session with a polluted context produces worse results. Knowing when to compact, when to start a new session and when to clean is essential to maintaining quality.

🎯Cleaning Rules

Rule 1: /compact at 70% context
When you feel that the answers are becoming generic or Claude starts to forget instructions, compact. Use/costto monitor context usage.
Rule 2: New session for new task
If you are going to completely change tasks (e.g. from building to debugging another module), start a new session. Residual context from another task is confusing.
Rule 3: /clear for visual focus
Use/clearwhen the screen is polluted but you want to maintain the context. Cleans the screen, maintains memory.
Rule 4: Compact with customized instruction
Use/compact "Mantenha foco em [tarefa X] e descarte discussoes sobre [Y]"to direct what to preserve.
