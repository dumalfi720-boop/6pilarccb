Topic 1: Overview
1

🏗️ Backend Project Overview

For a Python backend with FastAPI, the overview should focus onAPI and its consumers. Unlike a frontend, the context here is about endpoints, integration and data processing.

💡Complete Example

# Project Overview

DataPipe is a REST API backend for a data analytics platform.

It handles user authentication, data ingestion from multiple

sources (CSV, APIs, databases), transformation pipelines,

and serves processed data to a React frontend and mobile app.

The API serves ~2k requests/minute and processes batch jobs

of up to 10GB via background workers (Celery). All endpoints

are versioned under /api/v1/.

⚡Practical Tip

Mention the API consumers (React frontend, mobile app) so that Claude understands the full context. This influences decisions such as response format and endpoint versioning.

Topic 2: Tech Stack
2

⚙️ Tech Stack with FastAPI and SQLAlchemy

The Python stack has its particularities. It is essential to specify whether you useSQLAlchemy 1.x or 2.x, whether you use Pydantic v1 or v2, and which package manager (pip, poetry or uv).

💡Complete Example

#TechStack

- Language: Python 3.12

- Framework: FastAPI 0.109+

- ORM: SQLAlchemy 2.0 (async, new-style mapped_column)

- Validation: Pydantic v2 (model_validator, not validator)

- Database: PostgreSQL 16 + Redis 7 (caching/sessions)

- Migrations: Alembic

- Task Queue: Celery with Redis broker

- Auth: python-jose for JWT + passlib for password hashing

- Package Manager: uv (NOT pip, NOT poetry)

- Linting: ruff (replaces flake8, isort, black)

- Testing: pytest + pytest-asyncio + httpx (async test client)

- Containerization: Docker + docker-compose for local dev

- Deploy: AWS ECS with ECR

⚡Practical Tip

The "SQLAlchemy 2.0 (async, new-style mapped_column)" detail is crucial. Claude often generates SQLAlchemy 1.x code if not instructed. The same goes for "Pydantic v2 (model_validator, not validator)" - the APIs changed significantly between v1 and v2.

Topic 3: Commands with uv
3

🖥️ Commands with uv

O uvand the modern, ultra-fast Python package manager. If you use uv, it is essential to document the commands because Claude by default suggests pip or poetry.

💡Complete Example

#Commands

- `uv sync` - install/sync all dependencies from pyproject.toml

- `uv run uvicorn app.main:app --reload` - start dev server

- `uv run pytest` - run all tests

- `uv run pytest tests/path/to/test_file.py::test_name` - run single test

- `uv run pytest -x -v` - run tests, stop on first failure

- `uv run ruff check .` - lint check

- `uv run ruff format .` - auto-format code

- `uv run alembic upgrade head` - apply all migrations

- `uv run alembic revision --autogenerate -m "description"` - create migration

- `uv run alembic downgrade -1` - rollback last migration

- `docker-compose up -d` - start local PostgreSQL + Redis

- `uv add package-name` - add a new dependency

⚡Practical Tip

Note that all commands useuv runas a prefix. Without this documentation, Claude would usepython -m pytest ou pip installand would break the virtual environment.

Topic 4: Conventions
4

📐 FastAPI Conventions

The conventions of a FastAPI backend includeroute structure, Pydantic schemas, SQLAlchemy modelsand error handling standards. Each one directly impacts the quality of the generated code.

💡Complete Example

#CodeConventions

## API Structure

- Routers in app/api/v1/endpoints/ (one file per resource)

- Schemas (Pydantic) in app/schemas/ (Create, Update, Response)

- Models (SQLAlchemy) in app/models/ (one class per file)

- Business logic in app/services/ (never in routers)

- Database queries in app/crud/ (repository pattern)

## Pydantic v2 Patterns

- Use model_validator instead of validator decorator

- Use model_config = ConfigDict(...) instead of class Config

- Schema names: UserCreate, UserUpdate, UserResponse

## SQLAlchemy 2.0 Patterns

- Use Mapped[type] and mapped_column() (new style)

- Async sessions via async_sessionmaker

- Relationships with lazy="selectin" for async compat

##General

- All functions with type hints (params + return)

- Docstrings in Google format for all public functions

- snake_case everywhere (PEP 8)

⚡Practical Tip

The rule "Business logic in app/services/ (never in routers)" is fundamental. Without it, Claude tends to put all the logic directly into the routes, creating fat controllers that are difficult to test.

Topic 5: Things To Avoid
5

🚫 What to Avoid in FastAPI

The Python ecosystem hasspecific trapsthat Claude can fall down, especially in the difference between synchronous and asynchronous APIs, and between old and new versions of SQLAlchemy and Pydantic.

💡Complete Example

# Things To Avoid

- NEVER use SQLAlchemy 1.x patterns (Column(), declarative_base())

→ Use Mapped[], mapped_column(), DeclarativeBase

- NEVER use Pydantic v1 patterns (validator, class Config)

→ Use model_validator, ConfigDict

- DO NOT use sync database sessions (use async throughout)

- NEVER use pip install or poetry → always use uv

- DO NOT put business logic in route handlers

→ Create a service function in app/services/

- NEVER use print() for logging → use structlog

- DO NOT use datetime.utcnow() → use datetime.now(UTC)

- AVOID raw SQL queries → use SQLAlchemy ORM

- NEVER modify existing Alembic migrations

- DO NOT use wildcard imports (from module import *)

⚡Practical Tip

The rule "DO NOT use datetime.utcnow()" is a classic. This function is deprecated in Python 3.12, but Claude still suggests it out of habit. Include the correct alternative (datetime.now(UTC)) and essential.
