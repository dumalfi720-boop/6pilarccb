📄 What is CLAUDE.md

O CLAUDE.mdIt is a Markdown file that is at the root of your project and works as an "instruction manual" for Claude Code. It contains information about the project, code conventions, available commands and rules that Claude must follow when working on his code.

💡Main Concept

Think of CLAUDE.md as onboarding a new developer. Just as you would explain to a new colleague how the project works, CLAUDE.md does the same for Claude Code.

  • •And a simple text file in Markdown format
  • •It is versioned along with the code in the repository
  • •It is automatically read by Claude Code when starting a session

⚡Practical Tip

Create the fileCLAUDE.mdin the root of your project today, even if it starts with just 5 lines. A basic CLAUDE.md is better than none.

Topic 2
2

🤔 Why CLAUDE.md Exists

Without CLAUDE.md, Claude Code needsguesshow your project works. It doesn't know what frameworks you use, what code conventions to follow, or how to run tests. CLAUDE.md solves this by explicitly providing context.

💡Main Concept

The fundamental problem is that the Claude Code is powerful, but generic. He knows thousands of frameworks and patterns, but he doesn't know which one YOUR project uses. CLAUDE.md transforms Claude from a generic assistant into an expert in your project.

  • •Eliminates generic answers - Claude follows YOUR conventions
  • •Reduces errors - Claude knows what to avoid in your project
  • •Speeds up work - fewer manual corrections required

⚡Practical Tip

If you've already corrected Claude Code more than once about the same thing (e.g. "use Tailwind, not pure CSS"), this should be in your CLAUDE.md. Each repeated correction is a sign that information is missing in the file.

🧠 Project Memory Concept

CLAUDE.md works as thepersistent memoryof Claude Code between sessions. Unlike the conversation history that is lost, CLAUDE.md remains and is loaded automatically every time you start Claude Code in the project directory.

💡Main Concept

Each Claude Code session is independent - he doesn't remember what you talked about yesterday. CLAUDE.md is the solution for this: a file that persists between sessions and ensures that Claude always has the necessary context.

  • •Sessions are ephemeral, but CLAUDE.md is permanent
  • •Functions as Claude's "long-term memory" in his project
  • •The entire team shares the same context via version control

⚡Practical Tip

After a productive session where you taught Claude something new, add this information to CLAUDE.md before closing. So, in the next session, he will already know.

Topic 4
4

📖 When Claude Le o CLAUDE.md

Claude Code reads CLAUDE.mdautomatically at the beginning of each conversation. You don't need to ask it to read it - the file contents are loaded as part of the system prompt, even before your first message.

💡Main Concept

The reading mechanism is automatic and hierarchical. Claude searches CLAUDE.md files at multiple levels and combines them all to form the complete context.

  • •Automatic reading when startingclaudein any project directory
  • •Content enters context as high-level instructions
  • •Changes to the file during the session are not automatically reread

⚡Practical Tip

If you edit CLAUDE.md during a session, start a new conversation with/clearfor the changes to be loaded. Claude does not automatically reread the file in the middle of a session.

Topic 5

🎯 Concrete Benefits

Using CLAUDE.md bringsmeasurable benefitsin everyday life. The difference between a project with and without CLAUDE.md is like the difference between a GPS with and without the destination address - both work, but one gets there much faster.

💡Main Concept

The benefits accumulate over time. The more detailed and updated CLAUDE.md is, the better the results in each session.

  • •Consistency- All generated code follows the same standards
  • •Speed- Less time fixing and more time building
  • •Collaboration- Everyone on the team has the same "trained" Claude

⚡Practical Tip

Do an A/B test: use Claude Code in a project without CLAUDE.md and then add one. Compare the quality of the answers. You will notice the difference from the first interaction.

Topic 6

🚀 Getting Started

Starting with CLAUDE.md is simple. You can create amanuallyor use the command/initfrom Claude Code to generate one automatically. The important thing is to start - you can refine later.

💡Main Concept

The recommended approach is iterative: start with the basics and add information as you see Claude needs it.

  • •Create the fileCLAUDE.mdat the root of the project
  • •Start with: project name, main stack and 3 conventions
  • •Keep adding rules as you correct Claude repeatedly

⚡Practical Tip

Use the command/initwithin Claude Code to generate an initial CLAUDE.md based on the automatic analysis of your project. Then, customize it to your preferences.
