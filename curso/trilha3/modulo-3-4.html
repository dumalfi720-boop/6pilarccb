MODULE 3.4

👾 Understand a New Codebase

How to use Claude Code to quickly onboard any project, understanding architecture, patterns and flows in minutes.

1

🌍 Overview (Big Picture)

Start by asking Claude for aarchitecture overviewof the project. This gives you the map before exploring the details.

💡Main Concept

Questions to quickly understand the big picture:

  • •"Descreva a arquitetura geral deste projeto. Quais sao os principais modulos?"
  • •"Quais tecnologias e frameworks este projeto usa?"
  • •"Qual e a estrutura de pastas e o que cada pasta contem?"

⚡Practical Tip

Start simple:claude "O que este projeto faz? Me de uma visao geral em 5 pontos"

2

🔬 Zoom In

After understanding the big picture,dive into specific modulesthat you need to work on.

💡Main Concept

Explore specific areas with targeted questions:

  • •"Explique como o sistema de pagamentos funciona. Quais arquivos estao envolvidos?"
  • •"Como os dados fluem do frontend ate o banco de dados?"
  • •"Quais padroes de design sao usados neste modulo?"

⚡Practical Tip

Focus on the area where you will work:"Preciso mexer no modulo de usuarios. Me explique tudo sobre ele em detalhes"

3

🔄 Track a Stream

Ask Claude totrace a complete streamfrom end to end. This is the best way to understand how the pieces connect.

💡Main Concept

Trace request flows to understand the integration:

  • •"Trace o fluxo completo quando um usuario faz login: do click no botao ate a resposta"
  • •"Siga o caminho de um pedido: da criacao ate o envio do email de confirmacao"
  • •Claude will list each file and function in the order they are called

⚡Practical Tip

Piece with details:"Trace o fluxo incluindo: arquivo, funcao, linha, e o que cada passo faz"

4

📖 Use /init

The command/initgenerates a CLAUDE.md file with a summary of the project, standards and conventions that Claude will use in all future interactions.

💡Main Concept

/init creates persistent project memory:

  • •Enter/initat the root of the project
  • •Claude analyzes the codebase and generates a fileCLAUDE.mdwith: technologies, structure, patterns, useful commands
  • •In the next sessions, Claude reads CLAUDE.md automatically and already knows the context

⚡Practical Tip

Rotate/initas soon as you clone a new repo. Then edit CLAUDE.md to add details that you discovered manually.

5

❓ "Why?" Questions

The most valuable questions are those of"why"- they reveal design decisions, trade-offs, and historical context that no README explains.

💡Main Concept

“Why” questions to understand architectural decisions:

  • •"Por que usaram MongoDB ao inves de PostgreSQL neste projeto?"
  • •"Por que tem dois sistemas de autenticacao? Qual devo usar?"
  • •"Por que esta funcao e tao complexa? Tem algum edge case especial?"

⚡Practical Tip

Claude can analyze git log to infer decisions:"Olhe o historico de commits deste arquivo e me diga por que essa mudanca foi feita"

6

🚀 Rapid Onboarding

Combine all the techniques to makeonboarding in minutesinstead of days. Claude turns weeks of exploration into a conversation.

💡Main Concept

Complete onboarding guide in 5 steps:

  • •1. /initto generate CLAUDE.md
  • •2. Ask for an overview of the architecture
  • •3. Trace 2-3 main system flows
  • •4. Ask “why” questions about decisions you don’t understand
  • •5. Piece:"Quais sao as armadilhas comuns que um novo dev encontraria neste projeto?"

⚡Practical Tip

Finish with:"Se eu precisar adicionar uma nova feature de X, por onde eu comecaria? Quais arquivos preciso criar/modificar?"
