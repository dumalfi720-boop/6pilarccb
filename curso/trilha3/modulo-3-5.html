MODULE 3.5

👾 Write Tests

Workflow to automatically generate comprehensive tests, increase coverage, and practice TDD with Claude Code.

1

🔍 Identify Gaps

The first step andidentify what is not tested. Claude can analyze your codebase and pinpoint exactly where tests are missing.

💡Main Concept

Ask Claude to map the coverage:

  • •"Analise src/services/ e me diga quais funcoes nao tem testes correspondentes"
  • •"Rode npx jest --coverage e analise quais arquivos tem menos de 80% de cobertura"
  • •Claude compares the src/ structure with tests/ to find gaps

⚡Practical Tip

Start with the most critical files:"Quais sao os 5 arquivos mais importantes sem testes? Priorize por risco de bug"

2

🤖 Generate Tests

Ask Claude toautomatically generate tests. It analyzes the function, understands the inputs/outputs and creates comprehensive tests.

💡Main Concept

Claude generates end-to-end tests with edge cases:

  • •"Escreva testes unitarios para src/services/payment.js cobrindo todos os cenarios"
  • •"Inclua testes para: input valido, input invalido, erro de rede, timeout e permissao negada"
  • •Claude uses the testing framework that your project already uses (Jest, Mocha, Pytest, etc.)

⚡Practical Tip

Be specific in the scenarios:"Escreva testes para calculateDiscount() cobrindo: desconto zero, desconto maximo, cupom expirado, usuario premium"

3

✅ Check and Fix

After generating the tests,run them and fix faults. AI-generated tests may have issues that need tuning.

💡Main Concept

Verification and correction cycle:

  • •"Rode os testes que voce criou e corrija os que falharem"
  • •Claude analyzes the error, understands whether it is a problem in the test or in the code, and corrects it
  • •Repeat until everyone passes:"Continue rodando e corrigindo ate todos os testes passarem"

⚡Practical Tip

If a test fails, ask:"Este teste falhou. O problema e no teste ou no codigo de producao? Explique antes de corrigir"

4

📊 Coverage

Use metricscode coverageto guide where further testing is needed and measure progress.

💡Main Concept

Claude interprets coverage reports and acts on them:

  • •"Rode a cobertura e escreva testes para as linhas nao cobertas em auth.service.ts"
  • •"Aumente a cobertura de 60% para 90% neste modulo"
  • •Claude focuses on uncovered branches and lines, not redundant tests

⚡Practical Tip

Pipe the direct report:npx jest --coverage 2>&1 | claude -p "Analise esta cobertura e escreva testes para as areas descobertas"

5

🔁 TDD with Claude

PracticeTest-Driven Developmentusing Claude: write the tests first, then ask to implement the code that makes the tests pass.

💡Main Concept

The TDD cycle with Claude Code:

  • •RED:"Escreva testes para uma funcao validateEmail() que ainda nao existe"
  • •GREEN:"Agora implemente validateEmail() para fazer todos os testes passarem"
  • •REFACTOR:"Refatore a implementacao mantendo os testes passando"

⚡Practical Tip

TDD with Claude and powerful:"Quero criar um ShoppingCart. Escreva os testes primeiro definindo: addItem, removeItem, getTotal, applyDiscount. Depois implemente"

6

📚 Good Testing Practices

Follow thesegood practicesto ensure that your Claude-generated tests are actually useful and maintainable.

💡Main Concept

Principles for quality testing:

  • •Ask for descriptive names:"Use nomes de teste que descrevam o cenario: should_return_error_when_email_invalid"
  • •Avoid excessive mocks:"Prefira testes de integracao para os endpoints, mocks apenas para servicos externos"
  • •Each test should test one thing:"Garanta que cada it() testa apenas um comportamento"

⚡Practical Tip

Request a review of the tests:"Revise os testes que criamos. Tem algum fragil, redundante ou que testa implementacao ao inves de comportamento?"
