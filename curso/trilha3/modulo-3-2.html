MODULE 3.2

👾 Build New Feature

How to use Plan Mode to plan, implement and polish a new feature from scratch with Claude Code.

6
Topics
25
Minutes
Inter.
Level
Practical
Type
1

📋 Plan Mode - Planning

Before writing code, enable thePlan Modefor Claude to create a detailed feature plan. This avoids rework and guarantees a solid architecture.

💡Main Concept

Plan Mode makes Claude think before he acts:

  • •PressShift+Tabto switch to Plan Mode (or useTabto switch between Plan/Act)
  • •"Preciso de um sistema de notificacoes por email. Planeje a arquitetura"
  • •Claude will list files to create, dependencies, data flow and possible edge cases

⚡Practical Tip

Include specific requirements in the prompt:"Planeje um CRUD de produtos com: validacao, paginacao, filtros por categoria, e testes unitarios"

2

🔍 Review the Plan

Review and adjust the planbefore implementing. It's cheaper to change a plan than to refactor ready-made code.

💡Main Concept

Ask questions and ask for adjustments to the plan:

  • •"Use Redis para cache ao inves de in-memory"
  • •"Adicione rate limiting no plano"
  • •Check that the plan follows the standards of your existing project

⚡Practical Tip

Ask Claude:"Antes de implementar, analise o codebase existente e garanta que o plano segue os mesmos padroes de arquitetura"

3

🛠️ Implement

With the plan approved, move toAct Mode and implement. Claude will create files, write code and configure dependencies following the plan.

💡Main Concept

The implementation follows the step-by-step plan:

  • •PressTabto return to Act Mode and say:"Implemente o plano"
  • •Claude creates files, installs packages and configures everything automatically
  • •Review each file created - accept withyor ask for changes

⚡Practical Tip

For large features, implement in stages:"Implemente primeiro o modelo e a migration. Depois faremos o controller"

4

🧪 Test

Ask Claude towrite and run testsfor the new feature. Tests are an essential part of any feature.

💡Main Concept

Tests ensure that the feature works and does not break anything existing:

  • •"Escreva testes unitarios para o NotificationService"
  • •"Rode npm test e corrija qualquer falha"
  • •Request tests for edge cases: invalid inputs, permissions, limits

⚡Practical Tip

Use:"Escreva testes cobrindo: caso feliz, input invalido, usuario sem permissao, e limite de paginacao"

5

✨ Polish

Before the final commit, ask Claude topolish the code: improve names, add comments, clean imports and ensure consistency.

💡Main Concept

Polishing transforms functional code into professional code:

  • •"Revise o codigo que criamos. Melhore nomes de variaveis, adicione JSDoc e remova code smells"
  • •"Rode o linter e corrija qualquer warning"
  • •Check that there is adequate error handling at all points

⚡Practical Tip

Request a final review:"Faca uma ultima revisao de todos os arquivos que criamos. Tem algo que podemos melhorar?"

6

📦 Commit and Finalize

Finish with awell-structured committhat documents the new feature for the project history.

💡Main Concept

Well-made commits facilitate code review and history:

  • •Use/committo generate the message automatically
  • •For large features, consider atomic commits:"Faca commits separados: um para o modelo, um para o controller e um para os testes"
  • •Claude generates messages like:feat: add email notification system with queue support

⚡Practical Tip

For features that need PR, use/prright after commit. Claude creates the Pull Request with a complete description of the feature.
