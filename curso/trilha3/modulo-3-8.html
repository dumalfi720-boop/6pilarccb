MODULE 3.8

🪵 Debug with Logs and Errors

Learn how to use logs, stack traces, and error messages as direct input for Claude to quickly diagnose and fix problems.

📋 Pasting the Error Directly

The most direct way to debug with Claude andpaste the error directlyin the conversation. Claude analyzes the stack trace, identifies the problem file and line, and proposes a solution.

💡Main Concept

The simplest command to debug and start Claude with the error:

  • •claude "Estou recebendo este erro: [cole o stack trace]. Me ajude a corrigir."
  • •Always include the FULL stack trace, not just the final message
  • •Describe what you were doing when the error appeared

⚡Practical Tip

The more context you provide about when and how the error happens, the more accurate Claude's diagnosis will be.

2

🔄 Log Pipe for Claude

Instead of copying and pasting, you cansend logs directly via pipe. This is especially useful for large log files or extensive terminal output.

💡Main Concept

Use the pipe with the flag-pTo send direct content:

  • •cat error.log | claude -p "O que esta causando esses erros e como corrijo?"
  • •npm test 2>&1 | claude -p "Analise as falhas e sugira correcoes"
  • •docker logs app 2>&1 | tail -100 | claude -p "Diagnostique esses logs"

⚡Practical Tip

Usetail -100to limit the amount of log sent. Very large logs consume unnecessary tokens.

3

🔍 Tracing the Root Cause

Claude doesn't just read the error - hetraces the execution flowin your files to find the real root cause, not just the symptom.

💡Main Concept

After receiving the error, Claude automatically:

  • •Read the files mentioned in the stack trace
  • •Identifies the chain of calls that led to the error
  • •Distinguish between the symptom (where the error appears) and the cause (where the problem begins)

⚡Practical Tip

Ask Claude to explain the root cause before fixing: "Explain what is causing this before proposing the fix."

4

🔧 Applying the Correction

Once you have identified the root cause, ask Claude toapply the correction and checkthat the problem has been resolved.

💡Main Concept

The ideal prompt to apply the fix:

  • •"Corrija o bug e rode os testes para garantir que nada mais quebrou."
  • •Claude edits the file, runs the tests and confirms that the error is gone
  • •If new errors appear, it continues until everything is green

⚡Practical Tip

Always ask to run the tests after correction. Never accept a correction without automatic verification.

5

✅ Checking the Resolution

After correction, it is essentialverify that the error has actually been resolvedand that no side effects were introduced.

💡Main Concept

Post-fix verification checklist:

  • •The original error no longer appears
  • •All existing tests continue to pass
  • •The scenario that caused the error now works correctly

⚡Practical Tip

If the error was intermittent, mention it to Claude. It can suggest specific tests for race conditions or unexpected states.

6

🎯 Pro Tips for Debugging

Advanced tips to maximize machine efficiencydebugging with Claude Code.

💡Main Concept

Golden rules of efficient debugging:

  • •Include the FULL stack trace — never summarize
  • •Describe what triggered the error (user action, endpoint, etc.)
  • •Mention whether the error is intermittent or consistent
  • •Point to the suspicious file if you have an intuition

✓ What TO DO

  • ✓Paste the complete stack trace
  • ✓Describe the error trigger
  • ✓Ask for diagnosis before fixing

✗ What NOT to do

  • ✗Summarize the error in your own words
  • ✗Request fix without context
  • ✗Accept correction without running tests
