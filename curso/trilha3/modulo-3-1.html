MODULE 3.1

👾 Fix a Bug

The complete workflow for finding, diagnosing, and fixing bugs using Claude Code as your debugging partner.

Topics
20
Minutes

Level
Practical

1

🔍 Give Context to Claude

The first step to fixing a bug andgive enough contextfor Claude to understand the problem. The more information you provide, the faster it finds the solution.

💡Main Concept

Describe the bug clearly, including expected behavior vs. the current behavior. Use commands like:

  • •claude "O botao de login retorna erro 403 ao clicar. Deveria redirecionar para /dashboard"
  • •Include accurate error messages, stack traces and relevant logs
  • •Mention when the bug started (e.g. "after the last deploy") to help isolate the cause

⚡Practical Tip

Paste the error directly into the prompt:claude "Este erro aparece no console: TypeError: Cannot read property 'map' of undefined em UserList.jsx:42"

2

🕵️ Investigation

Leave Claudeinvestigate the codebaseto understand the full context of the bug. It will read files, follow imports and track the data flow.

💡Main Concept

Claude will automatically explore related files. You can guide the investigation:

  • •"Investigue o fluxo de autenticacao em src/auth/ para encontrar onde o 403 e gerado"
  • •"Verifique os middlewares que interceptam requests para /api/login"
  • •Claude uses grep, file reading and dependency analysis to map the problem

⚡Practical Tip

If Claude asks for permission to read many files, accept. A thorough investigation at the beginning saves time later. UseShift+Tabto accept all permissions at once.

3

🩺 Diagnosis

After the investigation, Claude will present hisdiagnosis with the root cause. It is important to validate this diagnosis before proceeding with corrections.

💡Main Concept

Validate the diagnosis by asking follow-up questions:

  • •"Tem certeza que o problema e no middleware e nao no controller?"
  • •"Existem outros lugares onde esse mesmo padrao pode causar bugs?"
  • •Ask Claude to explain the chain of events that leads to the bug

⚡Practical Tip

Ask Claude:"Antes de corrigir, me explique passo a passo como o bug acontece". This ensures that you understand the problem and can validate the solution.

4

🔧 Apply the Correction

With the diagnosis confirmed, ask Claude toimplement the correction. It will edit the necessary files and you can review each change.

💡Main Concept

Guide the correction with clear instructions:

  • •"Corrija o bug. Adicione validacao de null antes do .map() e um fallback para array vazio"
  • •Review the diff that Claude shows before accepting - pressyto accept ornto reject
  • •Ask to run the tests:"Rode npm test para garantir que nada quebrou"

⚡Practical Tip

Always ask to run tests after the correction:"Rode os testes relacionados e me mostre se passam". This prevents the fix from breaking something else.

5

📝 Correction Commit

With the bug fixed and tests passing, use the/commit commandto create a clean and descriptive commit automatically.

💡Main Concept

Claude generates commits following project conventions:

  • •Enter/commit- Claude analyzes the changes and creates a commit message
  • •It follows the Conventional Commits pattern:fix: resolve null pointer in UserList component
  • •You can request adjustments to the message before confirming

⚡Practical Tip

If you prefer more control, use:"Faca commit com a mensagem: fix: add null check before mapping users array". Claude runs git add and git commit for you.

6

🚀 Pro Tips

Advanced tips forspeed up debuggingand resolve more complex bugs efficiently.

💡Main Concept

Advanced techniques for efficient debugging:

  • •Direct log pipe:cat error.log | claude -p "Analise esses logs e encontre a causa do crash"
  • •Bug in production:"O Sentry mostra este erro em producao: [stack trace]. Encontre e corrija"
  • •Use/compactif the conversation gets long during debugging to keep the context clean

⚡Practical Tip

For intermittent bugs, ask:"Adicione logs temporarios nos pontos criticos para capturar o estado quando o bug ocorre". After reproducing, send the logs to Claude.
