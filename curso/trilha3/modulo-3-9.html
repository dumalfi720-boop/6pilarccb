🏗 Multi-file Refactoring

Master large-scale changes that affect multiple files—migrations, renames, and restructurings with security and testing at every step.

📋 Start with Plan Mode

Multi-file changes are the most risky. Always start atPlan Modeto map all affected files before touching any code.

💡Main Concept

The prompt to start a multi-file refactoring:

  • •claude "Preciso [migrar/atualizar/renomear] [X] em todo o codebase. Entre no plan mode."
  • •Claude will list all the files that need to change
  • •Review the plan before approving execution

⚡Practical Tip

Never do everything at once. Plan Mode prevents Claude from making changes without your approval.

2

🔍 Reviewing the Scope

Claude identifiesall affected files. Your role is to review whether the scope is complete and correct.

💡Main Concept

  • •Check that all relevant files are in the list
  • •Confirm that imports, tests and configurations are included
  • •Ask: "Are you sure we're not missing any files?"
3

📦 Run in Batch

The key to secure migrations:run in small batches, starting with the core module and moving to the dependents.

💡Main Concept

  • •"Comece pelo modulo core, depois mova para os modulos dependentes."
  • •Each batch must be independently testable
  • •Small commits between batches for easy rollback
4

🧪 Test After Each Batch

The golden rule:green tests at all times. Run the tests after each batch of changes.

💡Main Concept

  • •"Rode os testes apos cada lote de mudancas."
  • •If a test fails, stop and correct it before continuing
  • •Never accumulate batches without testing — the risk grows exponentially
5

🔎 Final Scan

After completing all batches, make afinal scanto ensure nothing is left behind.

💡Main Concept

  • •"Faca uma varredura final — tem algum arquivo que esquecemos?"
  • •Look for old references that may have escaped you
  • •Run the full test suite one last time
6

💨 Use /compact in Long Migrations

Long migrations consume a lot of context. The command/compactsummarizes the conversation and frees up space to continue.

💡Main Concept

  • •/compactsummarizes the entire conversation while maintaining the essential context
  • •Use between batches to keep the context window healthy
  • •Essential for migrations that take more than 30 minutes

✓ What TO DO

  • ✓Plan Mode before starting
  • ✓Small batches + testing
  • ✓Use /compact between batches

✗ What NOT to do

  • ✗Do everything at once
  • ✗Skip testing between batches
  • ✗Bypass Plan Mode
