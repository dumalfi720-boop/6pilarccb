🧭 Session Navigation - Moving Quickly

In a long session of Claude Code, you accumulate dozens of interactions on the screen. Scrolling with the mouse is slow and breaks the flow. Younavigation shortcutsallow you to jump between the beginning and end of the session, review previous outputs and return to the active prompt without taking your hands off the keyboard. Every second saved multiplies over hours of work.

Main Concept

🎯Main Concept

Claude Code runs inside your terminal, so all the terminal's standard navigation shortcuts work. Furthermore, Claude Code's own REPL adds specific behaviors for navigating the input history.

  • • Up Arrow / Down Arrow- Browse the history of prompts you entered. It works the same as bash history: up arrow brings you to the previous prompt, down arrow goes to the next one.
  • • Page Up / Page Down- Scrolls the screen up and down in large blocks. Ideal for reviewing long output without using the mouse.
  • • Ctrl+Home / Ctrl+End- Jump directly to the beginning or end of the session. When Claude generated 200 lines of code and you want to go back to the top to see the beginning, Ctrl+Home does the trick in one tap.

Practical Tip

💡Practical Tip

Keep your hands on the keyboard at all times. Each time you move your hand to the mouse, you lose between 1 and 3 seconds. In a 1 hour session with 50 interactions, this can add up to 2-3 minutes wasted just using the mouse. It seems like little, but over the course of weeks it turns into hours.

If your terminal does not respond to Page Up/Down (common in some emulators), check the terminal's keybindings settings. In iTerm2 (Mac), go to Preferences > Keys. In Windows Terminal, check the settings.json.

Do vs Avoid

✓ What TO DO

  • ✓ Use up/down arrows to navigate prompt history
  • ✓ Use Page Up/Down to review output without a mouse
  • ✓ Remember Ctrl+Home and Ctrl+End for big jumps

✗ What NOT to do

  • ✗ Use mouse to scroll when keyboard shortcuts exist
  • ✗ Retype entire prompts when up arrow brings back
  • ✗ Ignore terminal shortcuts just because "the mouse works"

✏️ Prompt Editing - Editing Before Sending

You typed a long prompt and realized there was an error in the middle. What it does? If the answer is "I delete everything and retype it", you are wasting time. Youreadline editing shortcutsallow you to navigate within the line, delete words, clear sections and reposition the cursor without using the mouse. They are standard Unix terminal shortcuts that work everywhere, not just in Claude Code.

Main Concept

🎯Main Concept

These shortcuts come from the libraryreadlinefrom GNU, used by bash, zsh, Python REPL, and virtually any command-line program. Learning these shortcuts once means you'll be faster in every terminal, not just Claude Code.

  • • Ctrl+A- Moves the cursor to the beginning of the line. When you need to add something to the beginning of the prompt.
  • • Ctrl+E- Moves the cursor to the end of the line. Return to the end after editing the beginning.
  • • Ctrl+W- Deletes the word before the cursor. Correct the last word without deleting everything.
  • • Ctrl+U- Clears everything before the cursor (or the entire line if the cursor is at the end). Quick reset.
  • • Ctrl+K- Deletes everything from the cursor to the end of the line. Keep the beginning and rewrite the ending.
  • • Alt+Backspace- Deletes the previous word. It works the same as Ctrl+W in most terminals.
Data/Research

Practical Tip

💡Practical Tip

The most powerful andSeta Cima + Ctrl+A + Ctrl+K. This retrieves the previous prompt, positions it at the beginning and erases everything to the end, giving a clean line based on history. Does it seem useless? In practice, you use it when you want to completely rewrite but starting from the right size of the previous prompt.

Practice these shortcuts in bash before using them in Claude Code. Open the terminal, type a long sentence and practice:Ctrl+A(start),Ctrl+E(end),Ctrl+W(delete word). In 10 minutes of practice, it becomes muscle memory.

Do vs Avoid

✓ What TO DO

  • ✓ Memorize Ctrl+A, Ctrl+E and Ctrl+W as a basic trio
  • ✓ Practice in bash beforehand to create muscle memory
  • ✓ Use Ctrl+U to quickly start over when you make a bad mistake

✗ What NOT to do

  • ✗ Hold Backspace to delete an entire line (use Ctrl+U)
  • ✗ Use side arrows to navigate word by word (use Ctrl+W)
  • ✗ Ignore these shortcuts thinking they are “nerd stuff”

⏹️ Execution Control - Pause, Cancel and Interrupt

Claude Code is generating a huge response that you have already seen is wrong. Or you realized halfway through the generation that you forgot to specify something. Wait for it to finish andwaste tokens and time. Execution control shortcuts are your handbrake: they stop generation, cancel operations and terminate sessions in a controlled manner.

Main Concept

🎯Main Concept

Each token generated has a cost. If Claude is writing 500 lines of code the wrong way, every line you let him generate is money wasted.Escape is the cheapest shortcut there isbecause it stops token bleeding instantly.

  • • Exhaust- Interrupts the current generation. Claude stops spawning immediately. You maintain what has already been generated and can give new instructions. This is the shortcut thatmore saves tokensin everyday life.
  • • Ctrl+C- Cancels the current operation and returns to the prompt. More aggressive than Escape. Use when you want to abort completely, not just pause the generation.
  • • Ctrl+D- Leave Claude Code's session. Sends an EOF (End of File) signal. The session is closed and you return to the normal terminal. Use when finished work.
Data/Research

📊Impact on the Token Economy

  • No Escape:Claude generates 2000 wrong answer tokens = ~$0.03-0.06 wasted per interaction
  • With Escape after 200 tokens:You save 90% of the tokens from that generation
  • In 50 interactions/day:If you stop 10 wrong answers early, you save $0.30-0.60 per day
  • In one month:~$9-18 savings just by pressing Escape at the right time
Alert

⚠️Attention

Ctrl+C vs Escape are different.Escape is gentle: for the generation but keeps the context intact. Ctrl+C is more crude: it can interrupt tool use operations in the middle (like writing a file). If Claude was in the middle of a Write operation, Ctrl+C could leave the file incomplete.

Simple rule: if Claude is just "talking" (generating text), useExhaust. If it is stuck in an operation that does not finish, then useCtrl+C.

Practical Tip

💡Practical Tip

Develop the squeeze reflexEscapeas soon as you realize that the answer is going in the wrong direction. Don't wait "to see if it gets better". In most cases, if the first 3-4 lines are wrong, the rest will be too. Stop, correct the prompt, and resubmit. This habit alone can cut your token consumption in half.

Do vs Avoid

✓ What TO DO

  • ✓ Press Escape immediately when the answer is wrong
  • ✓ Use Ctrl+C only for stuck operations
  • ✓ Use Ctrl+D to cleanly exit the session when finished

✗ What NOT to do

  • ✗ Wait for Claude to finish a wrong answer "so as not to lose"
  • ✗ Use Ctrl+C in the middle of a file write operation
  • ✗ Close the entire terminal to "cancel" (lose the session)

🔄 Command History - Reusing Previous Prompts

You wrote a detailed 5-line prompt asking for a specific refactoring. The result was almost right but needs an adjustment. Retype everything? Of course. THEcommand historyfrom Claude Code works the same as bash's history: arrow up retrieves previous prompts, and you can edit and resend. This transforms the iteration from minutes to seconds.

Main Concept

🎯Main Concept

Claude Code keeps a history of all the prompts you have typed in the session. This history is navigable with the arrows and editable with the readline shortcuts that you learned in the previous topic. The combination of these two features creates an extremely fast iteration flow.

  • • Up Arrow- Recovers the previous prompt. Press several times to go further back in history.
  • • Down Arrow- Advance in history (back to the present). If you went beyond the prompt you wanted, the down arrow brings you back.
  • • Edit and Resend- After recovering an up arrow prompt, use the editing shortcuts (Ctrl+A, Ctrl+E, Ctrl+W, etc.) to modify and press Enter to send the edited version.

Practical Tip

💡Practical Tip

Use history as an iteration tool. The ideal workflow is: send prompt > evaluate result > up arrow > adjust > resend. Each iteration refines the result without having to retype the basis of the prompt. In 3-4 iterations you arrive at the perfect result with minimal effort.

This works especially well for code review prompts, where you ask for incremental tweaks: "now also add logs", "now extract this function", "now rename to follow pattern X".

Do vs Avoid

✓ What TO DO

  • ✓ Use up arrow + edit to quickly iterate over prompts
  • ✓ Combine history with readline shortcuts for precise editing
  • ✓ Treat each iteration as incremental refinement

✗ What NOT to do

  • ✗ Retyping long prompts from scratch when up arrow resolves
  • ✗ Copy/paste previous prompt when history is faster
  • ✗ Ignore the partial result and redo everything from scratch

📝 Multi-line Input - Multi-Line Prompts

Single-line prompts work for simple commands. But when you need to passcontext, instruction and restrictionsAt the same time, just one line doesn't count. Claude Code's multi-line input lets you structure complex prompts across multiple lines, making them clearer for Claude and easier for you to maintain.

Main Concept

🎯Main Concept

By default, pressing Enter sends the prompt. To create a new line without sending, you need one of the multi-line input methods. The visual structure of the prompt directly influences the quality of Claude's response.

  • • Shift+Enter- Creates a new line without sending the prompt. This is the main and most intuitive method.
  • • Backslash (\) at the end- To add\at the end of the line indicates continuation, similar to bash. Useful when Shift+Enter does not work in your terminal.
Data/Research

🔍One Line vs Multi-line Prompt

One-line prompt (confusing, all together):

refatore auth.ts para usar async/await, adicione try/catch em cada operacao, mantenha a interface publica igual, adicione logs com winston e garanta que os testes continuem passando

Multi-line prompt (structured, of course):

Refatore o arquivo auth.ts:

Contexto: Preciso migrar de promises para async/await

Instrucoes:
- Converter todas as promises para async/await
- Adicionar try/catch em cada operacao async
- Adicionar logs com winston em cada catch

Restricoes:
- Manter a interface publica (exports) identica
- Garantir que os testes existentes continuem passando
Practical Tip

💡Practical Tip

Structure your multi-line prompts with a consistent pattern:Contextin the first part (what is happening),Instructionin the middle (what to do), andRestrictionsat the end (which cannot break). Claude processes structured prompts with much more precision than a wall of text.

Multi-line prompts are especially important for complex tasks like refactorings, migrations, and large code generations. The clearer the separation between what you want, how you want it and what you can't change, the better the result on the first try.

Do vs Avoid

✓ What TO DO

  • ✓ Use Shift+Enter to separate context, instruction and constraints
  • ✓ Structure complex prompts into clear visual blocks
  • ✓ Use lists with "-" for multiple instructions

✗ What NOT to do

  • ✗ Put 5 different instructions on a single giant line
  • ✗ Forgetting Shift+Enter and sending the prompt halfway
  • ✗ Use multi-line for simple one-sentence prompts (overengineering)

⚙️ Creating Custom Aliases - Your Shell Shortcuts

You typeclaude --resume10 times a day. Orclaude -p --model haiku "pergunta rapida"all the time. Each time 20-40 characters are typed unnecessarily.shell aliasesturn these repetitive commands into 2-3 letter shortcuts that you define once and use forever.

Main Concept

🎯Main Concept

An alias and shortcut that the shell automatically replaces with the full command. You define it in your shell configuration file (~/.bashrcfor bash,~/.zshrcfor zsh) and it is available in every new terminal.

  • • alias cr="claude --resume"- Resume the last session with just 2 letters. You use this every time you open the terminal to continue where you left off.
  • • alias cq="claude -p --model haiku"- Quick question with Haiku (cheaper and faster). Ideal for specific doubts that do not need Opus.
  • • alias crev="claude -p --model sonnet 'review this code'"- Instant code review with Sonnet. Pipe the file and receive feedback in seconds.

Code block: how to configure

💻How to Configure

Add these lines to your~/.zshrc(or~/.bashrc):

# Aliases for Claude Code
indeed cr="claude --resume"
indeed cq="claude -p --model haiku"
indeed cs="claude --model sonnet"
indeed co="claude --model opus"
indeed crev="claude -p 'review this code'"
indeed cex="claude -p 'explain this code'"

Then, reload the configuration:

source ~/.zshrc
Practical Tip

💡Practical Tip

The rule of thumb for creating an alias: if you type the same command more than 3 times a day, create an alias. Don't skimp on aliases, they cost nothing and each one saves seconds that add up. Keep aliases grouped at the end of your configuration file with a comment# Claude Code aliasesfor easy maintenance.

Pipe aliases are powerful. Example:cat arquivo.ts | crevsends the content of the file directly for review. Orgit diff | crevto review your changes before committing.

Do vs Avoid

✓ What TO DO

  • ✓ Create aliases for all the Claude commands you use daily
  • ✓ Use short and memorable names (cr, cq, cs, co)
  • ✓ Run source ~/.zshrc after editing to activate immediately

✗ What NOT to do

  • ✗ Create aliases with names that conflict with existing commands (ls, cd, etc.)
  • ✗ Forgetting to run source after editing the file (aka not active)
  • ✗ Create dozens of aliases that you won't remember (keep it simple)
