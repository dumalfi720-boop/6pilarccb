MODULE 5.10

⚠️ Skills Security

Skills and plugins have access to the filesystem, shell and environment variables. Learn how to check and secure your system before installing any skills.

6
Topics
15
Minutes
Advance.
Level
Critical
Type
1

📂 Filesystem Access Risks

Skills havefull access to the filesystem. A malicious skill can read, modify or delete any file that the user has permission to.

🚨Attention

Never install skills from unverified sources. Always read the source code before adding to your project.

2

💻 Shell Access Risks

Skills can performcommands in terminal. This means that a malicious skill can install software, modify settings or exfiltrate data.

💡Main Concept

  • •Check if the skill really needs to execute commands
  • •Look for suspicious commands like curl, wget, or send data
  • •Review/analysis skills generally DO NOT need a shell
3

🔑 Risks of Access to the Environment

Skills can accessenvironment variables, including tokens, API keys, and credentials stored in your shell.

💡Main Concept

  • •API keys, OAuth tokens and credentials are exposed
  • •A skill can read .env, .bashrc, .zshrc
  • •Use isolated environments to test unknown skills
4

📖 Reading Raw Markdown

Before installing any skill,read the full markdown file. Don't just rely on the description or name.

💡Main Concept

  • •Open the .md file in a text editor
  • •Read each instruction that will be followed by Claude
  • •Make sure it does exactly what it promises
5

🕵️ Checking Hidden Instructions

Malicious skills may containhidden instructions— prompt injection disguised in comments, whitespace or seemingly harmless sections.

💡Main Concept

  • •Look for instructions that do not correspond to the purpose of the skill
  • •Check for data exfiltration commands
  • •Be wary of skills that ask you to "ignore previous rules"
6

🔒 Plugin Security Review

For plugins, security review is even more important becauseplugins combine skills + MCPs + toolsin a single package.

✓ What TO DO

  • ✓Read the full source code
  • ✓Check reviews and reputation
  • ✓Test in an isolated environment

✗ What NOT to do

  • ✗Install without reading the code
  • ✗Trust the name only
  • ✗Give access to production credentials
