🔄 Synchronizing Skills between Code and Cowork

❓ The Synchronization Problem

Skills created inClaude Codestay in.claude/commands/as markdown files. NodeCoworking, skills appear as native slash commands. These two worlds do not automatically synchronize.

💡Main Concept

  • •Claude Code le skills de.claude/commands/
  • •Cowork has built-in skills + installed plugins
  • •There are 3 options to synchronize between the two
2

📁 Option 1: Select Skills Folder

The fastest way: in Cowork,select folderwhere your Claude Code skills are. Claude can read and follow them, but they do not appear as slash commands.

💡Main Concept

  • •In Cowork, select~/.claude/commands/
  • •Claude can read and follow instructions from .md files
  • •Fast but without native slash commands

⚡Practical Tip

Use this option when you need quick access without configuration. Ideal for testing skills before packing.

3

📦 Option 2: Package as Plugin

The best long-term solution: use/skill-creator ou /create-cowork-pluginin Cowork to transform your skills into a file.plugin.

💡Main Concept

  • •Plugins package skills + MCPs + tools in one file
  • •Install once, use for all projects
  • •Skills appear as native slash commands
4

🔨 Option 3: Recreate on Cowork

For simple skills,Ask Claude at Cowork to create the skill. It takes seconds and is formatted correctly for the Cowork environment.

💡Main Concept

  • •Ideal for short and simple skills
  • •Describe what the skill does and Claude creates it automatically
  • •Guaranteed native Cowork formatting
5

🛠 The Skill Creator

O /skill-creatorand the official tool for creating, testing and packaging skills professionally.

💡Main Concept

  • •Create skills with a standardized template
  • •Evals wheel to test quality
  • •Optimizes the description for better triggering
6

🎯 Best Practices

Rules for maintaining your skillsorganized and synchronizedbetween environments.

💡Main Concept

  • •Maintain a central skills repository
  • •Version your skills like any other code
  • •Use plugins for skills you use on multiple projects
  • •Keep project-specific skills in .claude/commands/
