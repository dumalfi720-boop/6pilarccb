💙 Skill: Screenshot to Website

One of the most impressive skills: transforming a design screenshot into functional HTML/Tailwind code. See how the skill analyzes layout, plans the structure and iterates until the result is pixel-perfect.

TOPIC 1

🎯 Purpose of the Skill

The "Screenshot to Website" skill receives an image (screenshot, mockup or wireframe) and generates HTML + Tailwind CSS code that visually reproduces the design. And the bridge betweendesign and implementation, eliminating hours of manual translation.

🎯Main Concept

# Purpose
Converter screenshot/mockup de interface em codigo HTML + Tailwind CSS
funcional e responsivo. Reproduzir o layout, cores, tipografia e
espacamento do design original com a maior fidelidade possivel.

The skill works with any type of image: screenshots of existing websites, Figma mockups exported as PNG, hand-drawn wireframes, or even prints of mobile apps that you want to replicate for the web.

💡Practical Tip

The higher the resolution of the screenshot, the better the result. Images of 1920x1080 or higher allow Claude to identify spacing details, fonts and icons with much greater precision. Avoid cropped or low-resolution screenshots.

TOPIC 2
2

🔍 Layout Analysis

The first step of the skill is to analyze the image anddecompose the layout into components. Claude identifies headers, sidebars, cards, grids, forms and other elements before writing a single line of code.

🎯Main Concept

  • •Visual hierarchy:Identify header, main content, sidebar, footer
  • •Grid system:How many columns, gaps, implicit breakpoints
  • •Color palette:Extract dominant colors, accents, backgrounds
  • •Typography:Sizes, weights, h1-h6 hierarchy, body text
  • •Repeated components:Cards, list items, buttons with the same style

💡Practical Tip

Add notes to the screenshot before sending. Text like "fixed navbar", "clickable card" or "dark mode" over the image helps Claude make better implementation decisions. Visual annotations save explanation tokens.

TOPIC 3
3

📐 HTML Planning

Before generating the final code, the skill creates aHTML structure plan. This ensures that the markup is semantic, accessible and organized before applying Tailwind styles.

🎯Main Concept

# Step 2: HTML Planning
Antes de escrever codigo, crie um outline da estrutura:

- nav > logo + menu items + CTA button
- main > hero section (h1 + subtitle + 2 buttons)
- main > features grid (3 columns, each with icon + title + desc)
- main > testimonials (carousel or grid)
- footer > 4 column links + copyright

Usar tags semanticas: nav, main, section, article, footer
Garantir acessibilidade: alt texts, aria-labels, roles

💡Practical Tip

Planning prevents the most common problem: spaghetti code with illogical nested divs. A good outline generates code that any dev can maintain. If the outline doesn't make sense, the code won't either.

TOPIC 4
4

🎨 Tailwind Build

With the layout analyzed and the HTML planned, the skill generates theComplete code using Tailwind CSS classes. Responsive by default, with sm/md/lg breakpoints and modern utilities.

🎯Main Concept

  • •CDN approach:Use Tailwind CDN for immediate results without build steps
  • •Responsive:Mobile-first with breakpoints for tablet and desktop
  • •Exact colors:Use arbitrary values ​​[#hex] when default Tailwind colors don't match
  • •Interactivity:Hover states, transitions, focus rings included

💡Practical Tip

Specify in the skill whether you want single-file HTML (all together with CDN) or separate components. For rapid prototyping, single-file is perfect. For real projects, ask for components that can be imported into your framework (React, Vue, etc.).

TOPIC 5
5

🔄 Comparison Loop

The skill includes avisual comparison stepwhere Claude compares the generated result with the original screenshot and makes adjustments. This feedback loop is what differentiates an “ok” result from a “pixel-perfect” result.

🎯Main Concept

# Step 5: Comparison Loop
1. Abrir o HTML gerado no navegador (ou visualizar mentalmente)
2. Comparar com o screenshot original:
   - Espacamentos estao corretos? (padding, margin, gap)
   - Cores batem? (background, text, borders)
   - Tipografia esta proporcional? (size, weight, line-height)
   - Layout responsivo faz sentido em mobile?
3. Listar diferencas encontradas
4. Aplicar correcoes
5. Repetir ate nenhuma diferenca significativa restar

💡Practical Tip

If you are using Claude Code with an MCP browser (like Puppeteer), the comparison loop can be automatic: Claude takes a screenshot of the result, compares it with the original, and iterates. Without browser MCP, you can paste the screenshot of the result for it to compare manually.

TOPIC 6
6

✅ Quality Checklist

The final checklist ensures that the generated code not only looks correct, but is alsofunctional, affordable and production-ready.

🎯Main Concept

  • ✓Valid HTML (no open tags, correct nesting)
  • ✓Responsive across 3 breakpoints (mobile, tablet, desktop)
  • ✓All images with descriptive alt text
  • ✓Minimum WCAG AA color contrast
  • ✓Interactions (hover, focus) on all clickable elements
  • ✓No custom CSS - 100% Tailwind utilities

💡Practical Tip

Add "100% Tailwind utilities, zero custom CSS" to the checklist if you want consistency with the rest of the project. Without this restriction, Claude can create separate CSS classes that break the Tailwind standard and make maintenance difficult.
