🌐 What MCP Browser Does

Main Concept

🎯Programmatic Web Browsing

MCP Browser gives Claude the ability to open real web pages, interact with page elements, take screenshots and extract content. It uses Puppeteer/Playwright behind the scenes to control a real Chrome browser in headless mode.

  • • navigate:Open any URL in the controlled browser
  • • click/fill:Click on elements and fill out forms
  • • screenshot:Capture screenshot of the current page (consumes a lot of tokens!)
  • • get_text/evaluate_js:Extract text from the page or run JavaScript
Why It Matters

📊The Only MCP With 'Eyes'

  • No other MCP allows visual UI verification
  • Can browse JavaScript-heavy websites (SPAs, React apps)
  • Interact like a real user: click, fill, scroll
Practical Tip

💡Practical Tip

Always prefer get_text over screenshot when possible. get_text returns pure text (few tokens) while screenshot sends an entire image (thousands of tokens). Use screenshot only when VISUAL verification is necessary - layout, colors, positioning.

Do vs Avoid

✓ What TO DO

  • ✓ Use get_text when you need textual content
  • ✓ Use screenshot only for visual verification
  • ✓ Disconnect after completing web tasks

✗ What NOT to do

  • ✗ Request screenshots 'to see how it is' without need
  • ✗ Keep Browser turned on during backend coding
  • ✗ Browse heavy websites unnecessarily
TOPIC 2
2

👑 Why and Elite

Main Concept

🎯Irreplaceable Power

Browser is classified as 'elite' because it offers a capability that NO other MCP replicates: visual and interactive access to the web. For visual QA, web scraping and deployment verification tasks, there is no alternative.

  • • Visual QA: checks if the deploy is rendering correctly
  • • Web Scraping: extracts structured data from any website
  • • Testing: tests user flows (login, checkout, forms)
  • • Competitive Intel: analyzes competitor websites
Why It Matters

📊When The Cost Is Worth

  • Valid when: the task REQUIRES viewing the web (QA, scraping, UI testing)
  • Not valid when: you only need code or API data
  • Rule: if you can do it without seeing a web page, you don't need a browser
Practical Tip

💡Practical Tip

The use case with the best ROI for Browser MCP and QA post-deploy: 'navigate to mysite.com and check that the header, footer and main page are correct'. A single command replaces 5-10 minutes of manual checking and catches visual bugs you might otherwise miss.

Do vs Avoid

✓ What TO DO

  • ✓ Use for visual QA after deploys
  • ✓ Use for scraping data not available via API
  • ✓ Evaluate ROI before connecting (high cost!)

✗ What NOT to do

  • ✗ Keep connected 'just in case' (very high cost)
  • ✗ Use for tasks that do not require web access
  • ✗ Relying only on screenshots without validating data with get_text

⚙️ Installation

Main Concept

🎯Setup with Puppeteer

Installation requires Chrome/Chromium installed on the system. MCP uses Puppeteer to control the browser in headless mode (without a visual interface). If Chrome is not available, the installation will fail.

  • • Prerequisite:Chrome or Chromium installed on the system
  • • Command: claude mcp add browser -- npx -y @anthropic/mcp-puppeteer
  • • Verification:Run /status and confirm that Browser appears as connected
  • • Test:Ask Claude 'navigate to google.com and say the title of the page'
Why It Matters

📊Installation Troubleshooting

  • If it fails: check if Chrome is installed (google-chrome --version)
  • On Linux without GUI: install chromium-browser (apt install chromium-browser)
  • In Docker: add --no-sandbox to puppeteer args
Practical Tip

💡Practical Tip

If you are on a server without a graphical interface (VPS, Docker), headless Chrome works normally but may need extra flags. Test with: npx -y @anthropic/mcp-puppeteer directly for errors before adding to Claude.

Do vs Avoid

✓ What TO DO

  • ✓ Check installed Chrome before installing MCP
  • ✓ Test with simple navigation after installation
  • ✓ Use headless mode (default) on servers

✗ What NOT to do

  • ✗ Install without Chrome/Chromium available
  • ✗ Ignore installation errors
  • ✗ Try using in environments without Chrome support

💼 Use Cases

Main Concept

🎯High-Value Workflows

MCP Browser turns Claude into a complete web agent. The most valuable use cases are those where visual verification or web interaction is mandatory and cannot be replaced by APIs.

  • • Visual QA:'navigate to localhost:3000 and check if the layout is correct'
  • • Web Scraping:'extract name and email from all companies listed on directorio.com/sp'
  • • Deploy Check:'open producao.com and confirm that the new feature is visible'
  • • Competitive:'compare the pricing of competitor-a.com vs competitor-b.com'
Why It Matters

📊ROI by Use Case

  • Visual QA: saves 5-10 min per deploy, catches visual bugs
  • Web Scraping: Saves hours of manual data collection
  • Deploy Check: automated confidence that the deployment worked
Practical Tip

💡Practical Tip

For web scraping at scale, combine Browser MCP with Google Sheets MCP: Browser extracts data from pages, Sheets stores it in an organized spreadsheet. Workflow: 'for each company in directorio.com/lista, extract name, email and telephone number, and save it in the Leads spreadsheet'.

Do vs Avoid

✓ What TO DO

  • ✓ Combine with MCP Sheets for scraping at scale
  • ✓ Use get_text for data and screenshot for visual verification
  • ✓ Disconnect immediately after web task

✗ What NOT to do

  • ✗ Scrape sites with aggressive anti-bot protection
  • ✗ Request screenshots of entire pages (focus on sections)
  • ✗ Use for tasks that don't involve the web
TOPIC 5
5

🔴 Token Impact: HIGH (~5,000+ tokens)

Main Concept

🎯The Most Expensive MCP

Browser MCP has the highest token tax in the ecosystem: ~5,000+ fixed overhead tokens per message, PLUS extra tokens for each screenshot or page content extracted. A single screenshot can cost an additional 10,000+ tokens.

  • • Fixed overhead: ~5,000 tokens per message (10x more than Filesystem)
  • • Screenshot: +5,000-15,000 tokens depending on resolution and complexity
  • • Page content (get_text): +1,000-5,000 tokens depending on size
  • • In 20 unused messages: 100,000 tokens wasted with only fixed overhead
Why It Matters

📊Why is it so expensive

  • Browser exposes 10+ complex tools with many parameters
  • Each tool has a long description so Claude can understand how to use it
  • Screenshots are base64 encoded images - very token heavy
Practical Tip

💡Practical Tip

MCP BROWSER GOLDEN RULE: only connect when you are going to use it, use what you need, disconnect immediately. The correct flow is: claude mcp add browser → use → claude mcp remove browser. NEVER leave Browser connected during coding sessions. The 5,000 tokens per message adds up quickly.

Do vs Avoid

✓ What TO DO

  • ✓ Connect ONLY when you need web access
  • ✓ Disconnect IMMEDIATELY after use
  • ✓ Monitor /cost while using the Browser

✗ What NOT to do

  • ✗ NEVER keep Browser 'always on'
  • ✗ NEVER forget Browser connected during coding
  • ✗ NEVER ask for unnecessary screenshots
TOPIC 6
6

🔌 Disconnect When Not UI

Main Concept

🎯Mandatory Operational Discipline

The most important rule of MCP Browser is: disconnect when not working on UI or web. Pure backend? Disconnect. Refactoring API? Disconnect. Writing tests? Disconnect. So need to check UI? Connect, check, disconnect.

  • • Connect: claude mcp add browser -- npx -y @anthropic/mcp-puppeteer
  • • To use:Do your web tasks (QA, scraping, testing)
  • • Disconnect: claude mcp remove browser
  • • Complete cycle:Less than 30 seconds between add → use → remove
Why It Matters

📊The Impact of Discipline

  • Browser turned on 30 unnecessary messages = 150,000 wasted tokens
  • Browser connected only 5 msgs needed = 25,000 tokens (savings of 83%)
  • The difference between professional and amateur operators lies in this discipline
Practical Tip

💡Practical Tip

Save the Browser add/remove commands in a shell alias: alias br-on='claude mcp add browser -- npx -y @anthropic/mcp-puppeteer' and alias br-off='claude mcp remove browser'. Thus, connecting and disconnecting becomes two characters + Enter. Zero friction to maintain discipline.

Do vs Avoid

✓ What TO DO

  • ✓ Create aliases for quick add/remove
  • ✓ Develop the reflex to disconnect after using
  • ✓ Treating the Browser as a one-off tool, not a permanent one

✗ What NOT to do

  • ✗ Leave Browser connected 'for later'
  • ✗ Forgetting to disconnect when changing tasks
  • ✗ Treat Browser as MCP that can stay 'always on'
