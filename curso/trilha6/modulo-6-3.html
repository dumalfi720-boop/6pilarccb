📂 MCP #1: Filesystem

📁 What Filesystem MCP Does

Main Concept

🎯Expanded File System Access

Filesystem MCP expands Claude Code's access beyond the directory where it was started. By default, Claude only reads and writes files within the current directory. With Filesystem MCP, it can access any path you authorize at the time of installation.

  • • read_file:Read contents of any file in authorized paths
  • • write_file:Write or create files in allowed paths
  • • list_directory:List contents of directories outside the project
  • • search_files:Search by pattern in files in other directories
Why It Matters

📊Why Expand Access

  • Copy configs and components between projects without leaving Claude
  • Access global templates that live outside the current repo
  • Work with monorepos or projects distributed across multiple directories
Practical Tip

💡Practical Tip

The MCP Filesystem is especially useful for those who work with multiple projects. Instead of closing Claude, navigating to another directory and reopening, you simply ask 'read the Button component from ~/projects/other-repo/src/components/Button.tsx'. Cross-repo without switching sessions.

Do vs Avoid

✓ What TO DO

  • ✓ Use to access files outside the current directory
  • ✓ Configure specific and limited paths
  • ✓ Always stay connected (low overhead)

✗ What NOT to do

  • ✗ Expose the system root (/)
  • ✗ Expose sensitive directories (~/.ssh, ~/.aws)
  • ✗ Use when files are in the current directory (unnecessary)

🏆 Why and High-Value

Main Concept

🎯Maximum Benefit, Minimum Cost

The Filesystem MCP has the best cost-benefit ratio of all MCPs: only ~500 tokens overhead per message for full access to any authorized directory. It is the only MCP where the recommendation is 'always keep it on'.

  • • ~500 overhead tokens = less than a paragraph of text
  • • Enables cross-repo workflow without switching sessions
  • • Very high ROI: negligible cost, constant benefit
Why It Matters

📊ROI comparison

  • Filesystem: 500 tokens → access to entire filesystem = HIGH ROI
  • GitHub: 2,000 tokens → access to issues/PRs = moderate ROI (only when working with GitHub)
  • Browser: 5,000 tokens → web access = variable ROI (high for UI, zero for backend)
Practical Tip

💡Practical Tip

Think of Filesystem as the 'MCP that pays its own cost'. The 500 overhead tokens are recovered the first time you avoid switching sessions to access another directory. From then on, pure profit in productivity.

Do vs Avoid

✓ What TO DO

  • ✓ Keep Filesystem always connected
  • ✓ Use as your 'fixed' MCP and rotate the others
  • ✓ Calculate time saved vs 500 tokens cost

✗ What NOT to do

  • ✗ Disconnect for token economy (negligible cost)
  • ✗ Treat with the same criteria as expensive MCPs (Browser)
  • ✗ Hesitate to install for fear of overhead

⚙️ Installation Command

Main Concept

🎯Quick Setup

Installation is a single command. The parameter after the package defines which directories Claude can access. You can specify multiple space-separated paths. Each path is a whitelist - Claude only accesses what you authorize.

  • • claude mcp add filesystem -- npx -y @anthropic/mcp-filesystem ~/projetos
  • • claude mcp add filesystem -- npx -y @anthropic/mcp-filesystem ~/projetos ~/configs
  • • Multiple paths: separate by space. Each is an authorized directory.
Why It Matters

📊Secure Configuration

  • Paths are WHITELIST: only what you list is accessible
  • Never use / (root) - exposes the entire system
  • Prefer specific paths: ~/projects, ~/dev, ~/work
Practical Tip

💡Practical Tip

Recommended command for most operators:claude mcp add filesystem -- npx -y @anthropic/mcp-filesystem ~/projetos. If you have global configs that you reference frequently, add:claude mcp add filesystem -- npx -y @anthropic/mcp-filesystem ~/projetos ~/.config. Check with /status after installing.

Do vs Avoid

✓ What TO DO

  • ✓ Use specific and limited paths
  • ✓ Check installation with /status
  • ✓ Test with 'list files in ~/projects'

✗ What NOT to do

  • ✗ Use / as path (exposes everything)
  • ✗ Expose ~/.ssh, ~/.aws, ~/.gnupg
  • ✗ Add unnecessary paths 'just in case'
TOPIC 4

💼 Practical Use Cases

Main Concept

🎯Day to Day Workflows

Filesystem MCP shines in cross-repo scenarios and global configuration access. It transforms Claude from 'stuck in the current directory' to 'system-wide agent'.

  • • Cross-repo:'copy the Header component from ~/projects/site-a to the current project'
  • • Global settings:'read my .gitconfig and suggest useful aliases'
  • • Templates:'use the template from ~/projetos/templates/api-endpoint.ts as a base'
  • • Comparison:'compare how authentication is implemented in ~/projetos/app-a vs ~/projetos/app-b'
Why It Matters

📊Real Productivity

  • Avoid switching sessions to reference other projects
  • Allows you to 'learn' from implementations of other repos of yours
  • Facilitates cross-project standardization by copying patterns
Practical Tip

💡Practical Tip

A powerful workflow: ask Claude to analyze how you solved a problem in another project and apply the same approach to the current one. 'Read how I implemented auth in ~/projects/app-saas/src/auth/ and implement something similar here'. Claude reads the other project and adapts it to the current context.

Do vs Avoid

✓ What TO DO

  • ✓ Reference implementations of other projects
  • ✓ Copy tested patterns between repos
  • ✓ Access global settings and templates

✗ What NOT to do

  • ✗ Use for files that are already in the current directory
  • ✗ Copy code without adapting to the context of the target project
  • ✗ Access other users' directories without permission

🟢 Token Impact: Low (~500 tokens)

Main Concept

🎯The Lightest MCP

With only ~500 tokens of overhead per message, Filesystem MCP is virtually invisible in context consumption. For perspective: 500 tokens is less than this section of text you are reading now.

  • • ~500 tokens = 0.25% of 200k context window
  • • In 50 messages: 25,000 total overhead tokens (acceptable)
  • • Comparison: Browser MCP costs 10x more (5,000 tokens)
Why It Matters

📊Numbers in Perspective

  • 500 tokens/msg and the cost of ~2-3 lines of code in context
  • To cause noticeable degradation, you would need 100+ MCPs like this
  • It is the only MCP where 'always on' is universally recommended
Practical Tip

💡Practical Tip

If someone suggests disconnecting the MCP Filesystem to save tokens, show the math: 500 tokens x 50 messages = 25,000 tokens. That's 12.5% ​​of a single compression. The value in cross-repo productivity exceeds this cost by orders of magnitude.

Do vs Avoid

✓ What TO DO

  • ✓ Stay connected without worrying about overhead
  • ✓ Use as a 'low cost' benchmark when comparing MCPs
  • ✓ Recommend as first MCP for new operators

✗ What NOT to do

  • ✗ Disconnect for savings (negligible cost)
  • ✗ Comparing overhead with heavy MCPs (doesn't make sense)
  • ✗ Hesitate to use because 'all MCP costs tokens'

✅ Security Best Practices

Main Concept

🎯Protecting Your System

Filesystem MCP accesses REAL files on your system. Careless configuration can expose Claude's credentials, SSH keys, and context-sensitive data. Security here is non-negotiable.

  • • NEVER:Expose / (root), ~/.ssh, ~/.aws, ~/.gnupg, ~/.env
  • • ALWAYS:Use specific paths like ~/projects, ~/dev
  • • ALWAYS:Review periodically with claude mcp list
  • • ALWAYS:Use the principle of least privilege - only what is necessary
Why It Matters

📊Real Risks

  • Exposed SSH keys can compromise all your servers
  • AWS credentials can generate astronomical costs if leaked
  • .env files with production secrets can compromise clients
Practical Tip

💡Practical Tip

After installing Filesystem MCP, run a security test: ask Claude 'try reading ~/.ssh/id_rsa'. If you can, you configured it wrong. Reconfigure with more restricted paths. The test takes 5 seconds and can prevent a security disaster.

Do vs Avoid

✓ What TO DO

  • ✓ Configure minimum and specific paths
  • ✓ Test by trying to access sensitive directories (should fail)
  • ✓ Periodically review authorized paths

✗ What NOT to do

  • ✗ Expose entire home (~/) for convenience
  • ✗ Ignore security 'because it's just my machine'
  • ✗ Add paths without thinking about what they contain
